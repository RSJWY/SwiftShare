# SwiftShare 虚拟共享空间方案（记录）

> 目标：将 SwiftShare 从“点对点发送”升级为“虚拟共享空间（临时网盘）”。

## 关键决策（已确认）

- 平台：**仅 Windows**
- 模式：**仅虚拟共享空间**（由接收端拉取，不再支持点对点推送）
- 列表同步：**mDNS/TXT 广播**（方案 B）
- 拖出下载：**使用系统临时目录**缓存文件，**程序关闭时清理**
- 预取策略：**仅小文件预取**（避免拖拽阻塞）
- 允许引入插件：**可使用 Tauri 插件实现拖出**

## 交互目标

- “拖入即共享，对方拖出即下载”
- 接收端列表展示为“虚拟文件夹/共享空间”

## 核心后端设计（Rust）

1) 共享索引（SharedRepo）
- 维护全局状态：`Arc<Mutex<HashMap<Uuid, FileMeta>>>`
- 文件拖入后生成 ID，记录：路径、文件名、大小、修改时间
- `get_shared_list` 提供当前虚拟共享列表

2) 拉取/流式下载
- 通过 HTTP 代理流或自定义 TCP 流读取远端文件
- 当接收端请求某 ID 时，实时读取并发送

3) 列表同步
- SharedRepo 变更后，通过 **mDNS TXT** 广播 FileManifest
- 前端接收广播事件并刷新 UI

## 拖出下载（Windows）

- 使用 **tauri-plugin-drag**（drag-rs）触发系统级拖出
- 仅支持**本地真实路径**：远端文件需先下载至临时目录
- 拖出流程：
  1. 前端 dragstart 触发请求
  2. 后端拉取远端文件到临时目录
  3. 调用 native drag 将本地路径交给资源管理器

## 预取策略（小文件）

- 仅在文件大小低于阈值时预取
- 大文件拖出时避免阻塞 UI，可显示“准备中/下载中”状态
- 具体阈值待定（需后续确认）

## UI 建议（React）

- 中心“篮子/气泡”区域展示虚拟共享
- 图标角标：
  - 我共享：上箭头
  - 他人共享：下箭头
- 动画：
  - 拖入：文件“跳入”列表
  - 拖出：文件图标闪烁/进度反馈
